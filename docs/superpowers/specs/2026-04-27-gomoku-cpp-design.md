# C++ 五子棋课程设计方案

## 目标

实现一个完全基于 C++17 标准库的控制台五子棋游戏系统，满足课程设计要求：欢迎与登录界面、说明界面、游戏界面、开始游戏、阅读游戏说明、按键功能说明、退出游戏、悔棋、复盘、按五子棋规则判断胜负并返回赢家信息。

## 平台约束

- 开发环境：macOS Apple Silicon。
- 运行环境：macOS 与 Windows。
- 构建系统：CMake。
- 语言标准：C++17。
- 依赖策略：只使用 C++ 标准库，不依赖 Qt、EasyX、ncurses 或平台专属 API。

## 功能范围

1. 欢迎界面显示游戏名称与主菜单。
2. 登录界面输入黑方和白方玩家姓名。
3. 说明界面展示五子棋基本规则。
4. 按键说明界面展示落子、悔棋、退出、复盘等操作方式。
5. 游戏界面显示 15x15 棋盘，玩家轮流输入坐标落子。
6. 悔棋功能撤销最近一步落子。
7. 复盘功能在游戏结束后按步骤回放整局棋。
8. 胜负判断支持横向、纵向、左斜、右斜连续五子。
9. 游戏结束后输出赢家姓名、棋子颜色与总手数。

## 架构设计

- `Board`：棋盘数据结构、坐标合法性、落子、撤销、胜负判断、棋盘渲染。
- `Player`：玩家姓名与棋子类型。
- `Replay`：记录每一步落子，支持清空、撤销最后一步、生成复盘棋盘。
- `Game`：组织登录、对局循环、命令处理、胜负返回与复盘入口。
- `UI`：菜单、说明、输入解析、终端输出。
- `main`：程序入口，启动主菜单。

## 数据流

玩家通过 `UI` 输入菜单选项或落子命令，`Game` 根据输入调用 `Board` 执行落子或撤销，并同步更新 `Replay`。每次合法落子后，`Board` 检查当前位置四个方向是否形成五连。游戏结束后，`Game` 根据结果输出赢家信息，并允许用户查看 `Replay`。

## 输入约定

- 主菜单使用数字选择。
- 对局中输入 `行 列` 落子，例如 `8 8`。
- 对局中输入 `u` 悔棋。
- 对局中输入 `q` 认输并返回主菜单。
- 棋盘坐标范围为 `1` 到 `15`。

## 错误处理

- 非数字菜单输入会提示重新输入。
- 越界坐标会提示重新输入。
- 已有棋子的坐标不能重复落子。
- 无棋可悔时提示不能悔棋。
- 输入空字符串或非法命令时提示格式错误。

## 测试策略

使用一个轻量级 C++ 测试程序验证核心逻辑：

- 横向五连胜利。
- 纵向五连胜利。
- 两种斜向五连胜利。
- 越界与重复落子失败。
- 悔棋后棋盘恢复为空。
- 复盘记录与撤销同步。
