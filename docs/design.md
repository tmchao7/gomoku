# Qt6 五子棋游戏系统设计文档

## 一、需求分析

本系统实现一个基于 C++17 + Qt6 Widgets 的图形界面五子棋游戏。玩家启动程序后先注册黑方和白方姓名，再选择普通模式或进阶模式。普通模式保持五连即胜；进阶模式在形成五连后进入选择消除、得分、转化对方棋子和连锁判定流程。系统支持悔棋、认输、重新开始，以及游戏结束后的自动复盘和手动复盘。

## 二、总体设计

系统采用“规则逻辑”和“图形界面”分离的结构：

```text
main -> BoardWidget -> Board
                    -> Types
                    -> Replay
```

`main.cpp` 只负责启动 Qt 应用；`BoardWidget` 负责绘制和交互；`Board` 负责棋盘状态与胜负判断。

## 三、模块设计

### 1. Board 棋盘模块

职责：

- 保存 15 x 15 棋盘状态。
- 判断坐标是否越界。
- 判断指定位置是否为空。
- 执行落子和撤销落子。
- 判断棋盘是否已满。
- 判断指定落子是否形成五连。
- 查找包含指定位置的五连候选。
- 根据两个端点校验刚好 5 颗连续己方棋子。
- 批量移除棋子。
- 将已有棋子替换为另一方棋子。

胜负判断方法：

每次落子后，只需要以该落子为中心检查四个方向：

- 横向：`(0, 1)`
- 纵向：`(1, 0)`
- 主对角线：`(1, 1)`
- 副对角线：`(1, -1)`

在每个方向上分别向正反两个方向统计连续同色棋子数量，总数大于等于 5 即获胜。

### 2. BoardWidget 图形界面模块

职责：

- 使用 `QPainter` 绘制棋盘线、星位和棋子。
- 使用 `QInputDialog` 输入黑方、白方玩家姓名。
- 使用 `QInputDialog` 选择普通模式或进阶模式。
- 使用 `mousePressEvent` 接收鼠标点击。
- 将鼠标坐标转换成棋盘行列。
- 调用 `Board::placeStone` 完成落子。
- 使用 `Replay` 记录落子历史，并支持悔棋。
- 提供悔棋、认输、复盘、重新开始按钮。
- 进阶模式中绘制可消除五连的彩色框线。
- 进阶模式中维护 `Playing`、`SelectingLine`、`ConvertingStone`、`GameOver` 状态。
- 游戏结束后进入复盘模式时，可以选择自动播放历史快照，或手动逐步查看。
- 黑白双方轮流切换。
- 调用 `Board::hasFiveInRow` 判断胜利。
- 使用 `QMessageBox` 弹窗提示赢家。

### 3. Types 公共类型模块

职责：

- 定义棋盘大小。
- 定义棋子类型：空位、黑子、白子。
- 定义游戏模式：普通模式、进阶模式。
- 定义落子记录结构。
- 提供棋子名称、棋子字符和对方棋子转换函数。

### 4. Replay 复盘模块

职责：

- 保存每一步落子记录。
- 支持撤销最后一步记录。
- 支持生成历史棋盘快照。

当前 GUI 版本已接入复盘按钮，游戏结束后可以自动播放历史棋局，也可以手动逐步查看棋盘状态。

## 四、关键交互流程

```text
启动游戏
  -> 输入黑方姓名
  -> 输入白方姓名
  -> 选择普通模式或进阶模式
  -> 黑方先手

鼠标点击
  -> BoardWidget::mousePressEvent
  -> 像素坐标转换为棋盘 row / col
  -> Board::placeStone(row, col, currentStone)
  -> Replay::addMove
  -> Board::hasFiveInRow(row, col)
  -> 若胜利，QMessageBox 提示“阵营 + 玩家姓名”
  -> 否则切换当前玩家

悔棋
  -> 读取 Replay 最后一步
  -> Board::removeStone
  -> Replay::undoLastMove
  -> 当前回合回到被撤销的玩家

认输
  -> 当前回合玩家认输
  -> 对方获胜
  -> QMessageBox 显示双方姓名和胜负结果

复盘
  -> Replay::snapshots 生成每一步棋盘快照
  -> 进入 replayMode
  -> 选择自动复盘或手动复盘
  -> 自动复盘由 QTimer 定时推进 replayStep
  -> 手动复盘使用上一步 / 下一步按钮切换 replayStep
  -> BoardWidget 绘制 replayBoard

进阶模式
  -> 当前玩家落子
  -> 若未形成五连，切换对方
  -> 若形成五连，绘制所有可消除五连
  -> 玩家点击同一条五连的两个端点
  -> 移除该五连并得 1 分
  -> 未达到 3 分时，玩家点击对方棋子转化为己方棋子
  -> 转化后若再次形成五连，继续选择消除
  -> 连锁结束后切换对方
```

## 五、构建设计

项目使用 CMake 管理构建：

- `gomoku_core`：棋盘规则、类型、复盘等核心逻辑。
- `gomoku`：Qt6 Widgets 图形界面可执行程序。
- `gomoku_tests`：核心规则测试程序。

Qt 依赖通过：

```cmake
find_package(Qt6 REQUIRED COMPONENTS Widgets)
```

引入，并链接到 GUI 目标：

```cmake
target_link_libraries(gomoku PRIVATE gomoku_core Qt6::Widgets)
```

## 六、测试设计

测试程序位于 `tests/test_core.cpp`，主要验证：

- 横向五连胜利。
- 纵向五连胜利。
- 两种斜向五连胜利。
- 越界落子失败。
- 重复落子失败。
- 悔棋后棋盘恢复为空。
- 复盘记录与撤销同步。
- 五连候选识别。
- 按端点选择合法五连。
- 批量移除棋子。
- 替换棋子颜色。
- 多条五连同时出现时都能识别。
