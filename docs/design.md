# 五子棋游戏系统设计文档

## 一、需求分析

本系统实现一个控制台五子棋游戏，包含欢迎与登录界面、说明界面和游戏界面。用户可以从欢迎菜单中选择开始游戏、阅读游戏说明、查看按键功能说明或退出游戏。对战过程中按照五子棋规则轮流落子，支持悔棋。游戏结束后输出赢家信息，并支持复盘查看整局落子过程。

## 二、总体设计

系统采用模块化设计，把棋盘规则、对局流程、复盘记录和终端界面分开实现。

```text
main -> Game -> UI
             -> Board
             -> Replay
             -> Player
```

## 三、模块设计

### 1. Board 棋盘模块

职责：

- 保存 15 x 15 棋盘状态。
- 判断坐标是否越界。
- 判断指定位置是否为空。
- 执行落子和撤销落子。
- 判断棋盘是否已满。
- 判断指定落子是否形成五连。
- 生成棋盘显示字符串。

胜负判断方法：

每次落子后，只需要以该落子为中心检查四个方向：

- 横向：`(0, 1)`
- 纵向：`(1, 0)`
- 主对角线：`(1, 1)`
- 副对角线：`(1, -1)`

在每个方向上分别向正反两个方向统计连续同色棋子数量，总数大于等于 5 即获胜。

### 2. Player 玩家模块

职责：

- 保存玩家姓名。
- 保存玩家棋子颜色。

黑方使用 `X`，白方使用 `O`。

### 3. Replay 复盘模块

职责：

- 保存每一步落子记录。
- 悔棋时删除最后一步记录。
- 游戏结束后按历史记录重新生成棋盘，实现逐步复盘。

### 4. UI 界面模块

职责：

- 显示欢迎菜单。
- 显示游戏规则。
- 显示按键说明。
- 获取玩家姓名。
- 显示棋盘和提示信息。

### 5. Game 对局控制模块

职责：

- 控制主菜单循环。
- 控制玩家登录。
- 控制黑白双方轮流落子。
- 处理悔棋、认输、非法输入。
- 在游戏结束时输出赢家信息。
- 调用复盘模块展示对局过程。

## 四、关键数据结构

棋盘使用二维数组：

```cpp
std::array<std::array<Stone, kBoardSize>, kBoardSize> cells_;
```

落子记录使用结构体：

```cpp
struct Move {
    int row;
    int col;
    Stone stone;
    std::string playerName;
};
```

## 五、跨平台设计

为了保证 macOS 和 Windows 都能运行，本系统：

- 使用 C++17 标准库。
- 使用 CMake 管理构建。
- 不使用 `system("cls")` 或 `system("clear")`。
- 不依赖 EasyX、Qt、ncurses 等额外库。
- 控制台显示采用普通 ASCII 字符，降低 Windows 乱码风险。

## 六、测试设计

测试程序位于 `tests/test_core.cpp`，主要验证：

- 横向五连胜利。
- 纵向五连胜利。
- 两种斜向五连胜利。
- 越界落子失败。
- 重复落子失败。
- 悔棋后棋盘恢复为空。
- 复盘记录与撤销同步。

## 七、总结

该设计结构清晰，核心规则和界面显示相互独立，便于测试、维护和答辩说明。项目难度符合本科 C++ 课程设计要求，同时保留了进一步扩展人机对战、文件保存复盘、排行榜等功能的空间。
